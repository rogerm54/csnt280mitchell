Objective 8.1 - Setting up a database backend for a web application

Throughout the course, you have seen how to set up PostgreSQL databases. In Project 1 and Project 2, you used a PostgreSQL database as the back end for a simple PHP/Apache web application. So, for this module, we are going to use a MongoDB database as our back end for a web application. In addition, we are going to use Node.js, and Node.js modules along with JavaScript to create the web application, instead of using PHP/Apache.

A simple MongoDB database

We are going to make a very simple MongoDB database so that it is easier to understand all the parts that go into a complete web application. This database will consist of a single collection called cars . If you have not already created and setup a Docker container that provides MongoDB, go back to this page and set that up.

✴ ✴ ✴ ✴ ✴

Here are the steps that I followed on my Apporto Virtual Desktop to create my Docker container that provides MongoDB: 


$ cd ~/Desktop/node_area/
$ mkdir -p ~/Desktop/node_area/docker_mongo/docs/cars
$ cd ~/Desktop/node_area/docker_mongo

I used the following Docker file, located in ~/Desktop/node_area/docker_mongo 


FROM mongo:4.2-bionic
WORKDIR /docs

The image and container were built using the following commands: 


$ cd ~/Desktop/node_area/docker_mongo
$ docker build -t mongo_img
$ docker container run -d --name-mongo_cont -v $(pwd)/docs:/docs mongo_img
$ docker stop mongo_cont

Open up two terminals that are in the ~/Desktop/node_area/docker_mongo/docs/cars directory. In one of the terminals, go into the Docker container using the following command: 


docker exec -it mongo_cont bash

Then, switch to the terminal that is in the host and create the file "init.js" as follows: 


$ geany init.js &

This is the contents for "init.js": 


conn = new Mongo()
db = conn.getDB("cars_db")
db.cars.drop()
db.cars.insert({ make: "Toyota", model: "Camry", price: 26123.99})
db.cars.insert({ make: "Honda", model: "Accord", price: 25137.99})
db.cars.insert({ make: "Chevrolet", model: "Camaro", price: 36423.99})

Line 1 connects to the Mongo server. Line 2 uses the database named cars . If cars_db does not exist, that database is created at this point. Line 3 drops the "cars" collection from the cars_db database. Lines 4-5 insert some documents into the "cars" collection.

To run "init.js", go to the terminal that is inside the Docker container: 


# mongo
> load("init.js")
true

To verify that the documents have been inserted, do a find() : 


> use cars_db
> db.cars.find()
{ "_id" : ObjectId("5dbdfbaf1cb39e39eb2700a0"), "make" : "Toyota", "model" : "Camry", "price" : 26123.99 }
{ "_id" : ObjectId("5dbdfbaf1cb39e39eb2700a1"), "make" : "Honda", "model" : "Accord", "price" : 25137.99 }
{ "_id" : ObjectId("5dbdfbaf1cb39e39eb2700a2"), "make" : "Chevrolet", "model" : "Camaro", "price" : 36423.99 }

The "cars" database has been set up.
