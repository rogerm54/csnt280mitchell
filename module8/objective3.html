Serving out our web application

To serve out our web application during development, we can make use of a HTTP server that is provided by the npm package, http-server. If this package is not already installed on your computer, you can install it using the following command: 


npm install -g http-server

The -g switch will install this package globally so that you can run http-server from the command line in any directory.

Obtaining Axios

When we set up the middleware, we implemented a set of REST routes that associate web URLs with functions that interact directly (through mongoose) with the back end database. That middleware exposes those functions through the routes to our front end. So, what we need is a way for our front end to communicate with those routes. This is where Axios comes in.

To get Axios, go to the following site, https://github.com/axios/axios/tree/master/dist and click on "axios.min.js". Click on the Raw table and save the file as "axios.min.js". Place this file inside the same directory (~/Desktop/node_area/docker_mongo/docs/cars) as your "index.js", "queries.js" and "cars_app.html" files.

Dealing with the CORS policy

CORS is an acronym for Cross-Origin Resource Sharing. A CORS error means that the browser has blocked resources coming from an origin outside the domain of the server that is serving up the web page. In this case, since we are running the web application from http://localhost:8000 , the Axios request is seen as coming from a different origin from the Express server (localhost:3000). One way to deal with this CORS problem is to install another Node.js package called cors .

Stop running "index.js". Make sure that cors has been installed by looking at this file, ~/Desktop/node_area/package.json . 


$ cat ~/Desktop/node_area/package.json

Creating the User Interface

Up until now, we have only been showing the results to the console. That is not going to work for an end user who probably does not even know the developer's console exists. At the same time, by showing the results in the console, you can focus on getting the functions to work before worrying about how to display the results in the window.

The main part of the user interface will be a HTML table that displays all the car documents. Part of this table is created with HTML markup and the rest of the table is created dynamically with JavaScript. This means that the HTML markup for the table must have an id attribute for the part that must be inserted dynamically. Here is the new version of "cars_app.html": 


<!DOCTYPE html>
<html>
   <head>
      <meta charset="utf-8">
      <script src="axios.min.js"></script>
      <script>
         document.addEventListener('DOMContentLoaded',init);
         let car_data = [];
         async function getCars() {
            try {
               const cars = await axios({
                  method: 'get',
                  url: 'http://localhost:3000/cars'
               });
               //console.log(cars.data);
               car_data = cars.data;
               updateCarsTable();
            }
            catch (error) {
               console.log(error);
            }
         }
         async function addCar(make, model, price) {
            try {
               await axios({
                  method: 'post',
                  url: 'http://localhost:3000/cars',
                  data: {
                     car_make: make,
                     car_model: model,
                     car_price: price
                  }
               });
               getCars();
            }
            catch (error) {
               console.log(error);
            }
         }
         async function updateCar(id, make, model, price) {
            try {
               await axios({
                  method: 'PUT',
                  url: `http://localhost:3000/cars/${id}`,
                  data: {
                     car_make: make,
                     car_model: model,
                     car_price: price
                  }
               });
               getCars();
            }
            catch (error) {
               console.log(error);
            }
         }
         async function getCarById(id) {
            try {
               const car = await axios({
                  method: 'get',
                  url: `http://localhost:3000/cars/${id}`
               });
               console.log(car.data);
            }
            catch (error) {
               console.log(error);
            }
         }
         async function deleteCar(id) {
            try {
               await axios({
                  method: 'DELETE',
                  url: `http://localhost:3000/cars/${id}`
               });
               getCars();
            }
            catch (error) {
               console.log(error);
            }
         }
         function removeChildren(elem) {
            while (elem.childNodes.length > 0) {
               elem.removeChild(elem.childNodes[0]);
            }
         }
         function updateCarsTable() {
            const cars_tbody = document.getElementById('cars_tbody');
            removeChildren(cars_tbody);
            for (let i = 0; i < car_data.length; i++) {
               const car = car_data[i];
               let id = car._id;
               let make = document.createTextNode(car.make);
               let model = document.createTextNode(car.model);
               let price = document.createTextNode(car.price);
               let tr = document.createElement('tr');
               tr.setAttribute('id',id);
               let td = document.createElement('td');
               td.appendChild(make);
               tr.appendChild(td);
               td = document.createElement('td');
               td.appendChild(model);
               tr.appendChild(td);
               td = document.createElement('td');
               td.appendChild(price);
               tr.appendChild(td);
               cars_tbody.appendChild(tr);
            }
         }
         function init() {
            getCars();
         }
      </script>
   </head>
   <body>
      <h1>Cars database application</h1>
      <table border="1">
         <caption>Cars Table</caption>
         <thead>
            <tr>
               <th>make</th> <th>model</th> <th>price</th>
            </tr>
         </thead>
         <tbody id="cars_tbody"></tbody>
      </table>
   </body>
</html>

Lines 65-81 define the showEditDialog() function. This function is declared as async on line 65, so that the await operator can be used on line 68. Line 66 is commented out, but was used to show the value of the currentTarget property of the event that caused showEditDialog() to be called. Line 67 gets the id property of the <tr> element that was clicked on to trigger this event. This means that the id value comes from that table row that was clicked on. This is what allows the user to select which car document to edit. Whichever row for the car is clicked on, that will be the car document selected for editing.

This is a very important part of the user interface. Handling the click event associated on clicking with a row in the cars table allows choosing the car to be edited. This is where JavaScript shines because it makes the user interface much cleaner. If we use PHP to provide the user interface, we would need to add radio buttons next to each car row, to allow selection of the car to be edited. This is clumsier and requires multiple pages to carry out the editing process.

Another important thing to note about line 67. When you use a click event generated by an element besides a button, you need to take into account what that element is. There is a difference between event.target and event.currentTarget . When the user clicks on a table row, that row consists of several <td> elements inside a <tr> element. If you click on the make of the car, event.target will be the <td> element holding the make. But, event.currentTarget will be the enclosing container, the <tr> element. Since the id of the car is tied to the <tr> element on line 27, we need to use event.currentTarget (and not event.target).

Getting back to the showEditDialog() function. Line 68 uses the await operator to make sure that execution of the code stops until the car document is returned by the getCarById() function. Lines 69-71 get references to the <input> elements for the make, model and price, respectively. Lines 72-74 set the values of those text boxes to be the values from the car document that is to be edited. This makes it so that the user has to enter as few things as possible to edit the car document. Lines 75-80 just cause the edit dialog box to display, but with the textboxes set with the existing car document's values.
