Objective 5.1 - Using scripts to process data files into SQL insert statements

A common problem that people who work with database applications run into is having to take existing data and put it into a database. Sometimes the data was stored in an Excel spreadsheet. In other cases, the data may have come from an existing database application and is in the form of a comma-separated-value (CSV) file.

In an ideal world, your organization bought the perfect database application that you can use forever. In practice, this does not happen. So, your organization is going to have to move existing data into a new database application at some point. If you are lucky, the existing data is already in a format that can be directly imported into a table in the new database. However, most likely you will have to do some manipulation of the data to get the data into a format that can be imported.

CSV files for a single table

Let's start with a case that the existing data is a CSV file that should be imported into a single table. Suppose that we have some course data that is for courses in the CSNT program. Here is the file "courses.csv": 


prefix,number,title,credits,location
CENT,110,Intro to Information Systems,3,2-608
CENT,132,ICT Support,4,20-15
CENT,140,Computer Networking I,4,13-104
CENT,228,Sys Adm & TCP/IP,4,13-112
CENT,231,Telecommunications,4,20-11
CENT,240,Computer Networking II,4,13-104
CENT,270,Network Operating Systems I,4,13-108
CENT,275,Security Essentials,3,13-112
CENT,280,Database Systems I,3,2-603
CENT,310,Network Security,3,13-108
CENT,315,Network Management,3,13-112
CENT,330,Ethical Hacking,3,13-108

To keep things simple, assume that we only have one course prefix. In reality, you would have many course prefixes (at least one for each department on campus). We have two things that we would like to change for our new database. In the new database, we want to have a table called courses and another table called sections . This is because for several courses, there can be more than one section. So, we don't want to store the location with the course information. The location would be stored with the section information. The other thing we want to change is that the prefix should be changed to 'CSNT' instead of 'CENT'.

With this small amount of data, we could easily hand enter the new data in for our courses table. But, let's come up with a solution that would scale if we had a CSV file with 2000 rows instead of 13. Assume that we have our courses table that would be created using the following SQL statement: 


create table courses(
   id serial,
   prefix text,
   number text,
   title text,
   credits integer,
   primary key (id)
);

This could be processed using a Python program like this: 


# make_courses.py
infile = open("courses.csv","r")
infile.readline() # skip header line
lines = infile.readlines()
infile.close()
for line in lines:
   tokens = line.strip().split(",")
   prefix = tokens[0].replace('E','S')
   print('insert into courses (prefix, number, title, credits) values ')
   print(" ('" + prefix + "','" + tokens[1] + "','",end='')
   print(tokens[2] + "'," + tokens[3] + ");")

$ python3 make_courses.py > add_course_data.sql

That is, just use the redirection operator, > , to output to that SQL script.

Using a Node.js solution

JavaScript is becoming a more important language with respect to databases. There has been a definite movement from web applications served out using Apache and PHP, to web applications that use Node.js. So, it is worth gaining some familiarity Node.js and JavaScript.

Let's solve the problem using a Node.js package called fs . The virtual machines already have Node.js and npm the Node package manager installed. If you want to install this on another Ubuntu machine, you can follow the steps from the virtual machine setup Once you have Node.js and npm installed, you can start by creating a working directory on your Desktop: 


$ mkdir -p ~/Desktop/node_area/courses_test
$ cd ~/Desktop/node_area/courses_test

Once you are in that directory, run the following command: 


$ npm init -y

The first line initializes the directory so that Node.js packages can be set up to be local to that directory. That instruction creates a subdirectory called node_modules that will be used to hold any Node.js packages you install. That instruction also creates a file called "package.json" that will hold the configuration that Node.js needs to manage any packages you install.

Next, run this command: 


$ npm install fs

This will install the fs package inside the node_modules directory set up by the previous command. Now, you can start using JavaScript code that uses the fs package. Here is the starting code for the file "make_courses.js": 


const fs = require("fs");
fs.readFile("courses.csv", function(err, buff) {});

Line 1 is used to import the fs module from the fs package. Line 2 uses the fs.readFile() function to read the file "courses.csv". On line 2, the second argument to the fs.readFile() function is an anonymous function with two parameters. The first parameter, err , is an object that holds any error message. The second parameter, buff , is an object that holds the contents of the file being read. That function is not doing anything yet, as the curly braces ( { } ) surrounding the body of that function are empty at this point.

Let's make a small change to the code, so that it is easier to see that we have not yet defined the body of the anonymous function that handles the event that the file is read. Here is the new code: 


const fs = require("fs");
fs.readFile("courses.csv", function(err, buff) {
   
});

Now, it is easier to see that line 3 is just the empty body of that anonymous function. Before modifying the code further, make sure you copy "courses.csv" into the working directory (~/Desktop/node_area/courses_test). Then, this is the new version of the code: 


const fs = require("fs");
fs.readFile("courses.csv", function(err, buff) {
   let data = [];
   data = buff.toString().split("\n");
   console.log(data);
});

The new lines are lines 3-5. On line 3 we declare data to be an empty array. On line 4 we use the toString() method to get the contents of the read file into string format and split that string using a newline ( \n ) character. The resulting array is assigned to data . Line 5 just prints the data array to the console. To run this program, use the following command: 


$ node make_courses.js

const fs = require("fs");
fs.readFile("courses.csv", function(err, buff) {
   let data = [];
   data = buff.toString().split("\n");
   for (let i = 1; i < data.length - 1; i++) {
      console.log(data[i]);
   }
});

The new lines are lines 5-7. These lines setup a for loop that goes over all the elements of the data array except for the first and last element. Line 6 just prints out the element to the screen. Here is the result of running the code now:

At this point, what we have to do is very similar to the Python program. Here is the new version of the code: 


const fs = require("fs");
fs.readFile("courses.csv", function(err, buff) {
   let data = [];
   data = buff.toString().split("\n");
   for (let i = 1; i < data.length - 1; i++) {
      let tokens = data[i].trim().split(",");
      const prefix = tokens[0].replace('E','S');
      let string = "insert into courses (prefix, number, title, credits)";
      string += " values\n ('" + prefix + "','" + tokens[1];
      string += "','" + tokens[2] + "'," + tokens[3] + ");";
      console.log(string);
   }
});

The new lines are lines 6-11. Line 6 calls trim() to trim off any whitespace and then calls split() to split the trimmed string using a ",". The resulting array is stored as tokens . Line 7 takes the first token and converts 'CENT' to 'CSNT'. Line 8 starts defining a variable called string that is the beginning of the SQL INSERT statement that we need. Lines 9-10 just complete that string by inserting in the appropriate values from the tokens array. Finally, line 11 prints out that string. Here is the result of running this program:

A web-based solution

What if we wanted to solve this in a web browser, without the use of Node.js? Once you get used to the syntax, the Node.js solution using the fs package is really not much different than the Python solution. But, if we don't use Node.js and try to solve this in a web browser, that is a different situation.

Start with the following code, that was developed on this page : 


<!DOCTYPE html>
<html>
   <head>
      <meta charset="utf-8">
      <script>
         document.addEventListener('DOMContentLoaded',init);
         function readUploadedFile(file) {
            const fileReader = new FileReader();
            let promise = new Promise((resolve, reject) => {
               fileReader.onerror = () => {
                  fileReader.abort();
                  reject(new DOMException("Problem reading"));
               };
               fileReader.onload = () => {
                  resolve(fileReader.result);
               };
               fileReader.readAsText(file);
            });
            return promise;
         }
         async function handleLoadFile() {
            const chooseFile = document.getElementById('chooseFile');
            const file = chooseFile.files[0];
            let contents = await readUploadedFile(file);
            processFile(contents);
         }
         function processFile(contents) {
            // This is the function that you modify
            let data = contents.split("\n");
            let out = "";
            for (let i = 0; i < data.length; i++) {
               out += data[i] + "\n";
            }
            const textarea = document.getElementById('textarea');
            textarea.value = out;
            let out_data = new Blob([out], {
               type: 'text/plain'
            });
            let url = window.URL.createObjectURL(out_data);
            const download = document.getElementById('download');
            download.href = url;
         }
         function init() {
            const loadFile = document.getElementById('loadFile');
            loadFile.addEventListener('click',handleLoadFile);
         }
      </script>
   </head>
   <body>
      <input id="chooseFile" type="file">
      <button id="loadFile">Load File</button><br>
      <a id="download" href=""
         download="out.txt">Download output</a><br>
      <textarea id="textarea" rows="5" cols="80"></textarea>
   </body>
</html>

So, you now have three options for processing data files to create a SQL script. You could do this processing with a Python script, a Node.js script (using the fs module), or a web-based JavaScript solution. The method you choose will depend on which language you prefer using. The main advantage of the web-based solution is that it is a GUI program that lets you browse for the input file and also for the output file location and name. The disadvantage with the web-based solution is that you need to serve out the page using the http-server package.
