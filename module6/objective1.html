Objective 6.1 - Creating tools that can automate the running of SQL scripts on a relational database

If you need to manage several databases, it might be useful to be able to manage them without having to use a command line tool like psql . A GUI program that could be used to connect to a PostgreSQL database and run SQL scripts could be useful because you would not have to remember the different psql commands. You can also create a program that would make it easy for someone to use, even if they did not know SQL very well.

Using python3-psycopg2 from a command line program

Before we write a GUI program, let's write a command-line program just to get used to some of the things that can be done making use of the python3-psycopg2 module. Here is a program that creates a database called test_db that will be owned by the 'bob' user: 


"""
create_db.py A program that demonstrates how to create a 
database using the psycopg2 module.
"""
import psycopg2
conn_string = "host='localhost' dbname='csnt280db' "
conn_string += " user='csnt280man' password='pgsql_man'"
conn = psycopg2.connect(conn_string)
conn.autocommit = True
cursor = conn.cursor()
sql = "drop database if exists test_db;"
cursor.execute(sql)
sql = "create database test_db owner bob;"
cursor.execute(sql)
conn.close()

Set up so that you have three terminals open. Two terminals should be inside the Docker container that has Python and psycopg2 installed, and the other terminal should be in a directory inside of the <main_docker>/docs directory. That third terminal is where you create the "create_db.py" file. After creating that file, use one of terminals inside Docker to change to the location of your "create_db.py" file. Run the script. In the other terminal inside the Docker container, use psql to verify that the "test_db" database exists and can be logged into by 'bob'.

The program is mostly straightforward. Line 5 imports the psycopg2 module. Lines 6 and 7 are used to create the connection string that psycopg2 will use to connect to the PostgreSQL database. That string is broken over two lines to keep the length of the instructions relatively short. Line 8 attempts to get the connection and store the connection in conn . Although you could try to catch the exception that could be thrown, for a simple program like this, it is better for the exception to be thrown so that the program ends and you can see the error.

Line 9 sets the connection's autocommit property to True. This is required to drop or create a database. If you don't have this line, trying to execute the query on line 12 and/or line 14 will result in an error stating that those kinds of commands are not permitted in a transaction. Line 11 sets up a drop query that will drop the "test_db" database if it exists. Line 12 runs that query. So, if the "test_db" database exists, it will be dropped. Otherwise, the drop command is ignored. Line 13 sets up a create query that will be used to create the "test_db" database. Line 14 executes the create query and the "test_db" database should be created. Line 15 just closes the connection to the PostgreSQL server.

Using one of the terminals inside the Docker container, verify that you can log into the "test_db" database as 'bob'.

Creating SQL scripts for "test_db"

Here is a the script "init.sql" that we can use to initialize the tables and a view for "test_db": 


-- init.sql for the test_db database
drop table if exists students cascade;
drop table if exists sections cascade;
drop table if exists students_sections cascade;

create table students(
   id serial,
   first_name text,
   last_name text,
   primary key (id)
);
create table sections(
   id serial,
   course_alpha text,
   credits integer,
   primary key (id)
);
create table students_sections(
   id serial,
   student_id integer references students(id),
   section_id integer references sections(id),
   primary key (id)
);
create view students_view as
   select students.id as st_id, students.first_name,
   students.last_name, sections.id as sec_id,
   sections.course_alpha, sections.credits
   from students join students_sections
   on students.id=students_sections.student_id 
   join sections on 
   students_sections.section_id=sections.id;

Here is a console-based Python program, "initialize_db.py" that will be used to call both of those scripts: 


# initialize_db.py
import psycopg2
conn_string = "host='localhost' dbname='test_db' "
conn_string += " user='bob' password='somepass'"
conn = psycopg2.connect(conn_string)
cursor = conn.cursor()
infile = open("init.sql","r")
sql = infile.read()
infile.close()
cursor.execute(sql)
sql = open("add_initial_data.sql").read()
cursor.execute(sql)
conn.commit()
conn.close()

Lines 3-5 are used to get a connection to the database "test_db" using the 'bob' user. Line 6 obtains a cursor to allow execution of SQL commands. Line 7 opens the "init.sql" file. Line 8 reads the entire contents of that SQL script and save it as the sql variable. Line 10 executes the SQL commands from "init.sql". Line 11 gets the contents from "add_initial_data.sql" and stores the contents inside the sql variable. Line 12 causes those statements to run. Line 13 commits the changes to the database. If you failed to make this commit, the database tables would not be created and the data would not have been inserted into those tables. Using tkinter to create a GUI program

Now that we know how to run SQL scripts from a Python console program, let's look at writing a GUI program that makes use of the tkinter module. Here is the program, "query_db.py": 


"""
To install tkinter on UbuntuMate 20.04
sudo apt install python3-tk
"""
import tkinter as tk

class MyGui(tk.Tk): # inherit from Tk class in tk module
   def __init__(self):
      tk.Tk.__init__(self) # call parent class constructor
      self.title('Database Query Tool')
      self.geometry('700x600+50+50') # 700x400 at 50,50

app = MyGui()
app.mainloop()

This will just put up a very simple GUI application that consists of a window that is 700 pixels wide by 600 pixels in height. The title bar will show the text "Database Query Tool". To close the application, you click on the close button.

Let's add a menu to the application so that we can quit the application by selecting a menu item. Here is the new code: 


"""
To install tkinter on UbuntuMate 20.04
sudo apt install python3-tk

import tkinter as tk

class MyGui(tk.Tk): # inherit from Tk class in tk module
   def __init__(self):

      self.initMenu()

   def initMenu(self):
      menubar = tk.Menu(self)
      fileOptions = tk.Menu(menubar)
      menubar.add_cascade(label='File',menu=fileOptions)
      fileOptions.add_command(label='Quit',command=self.destroy)
      self.config(menu=menubar)
      
app = MyGui()
app.mainloop()

Line 10 calls the initMenu() function that is defined on lines 14-19. Line 15 creates a menu bar. Lines 16 and 17 create the File menu so that fileOptions will hold the menu items for the File menu. Line 18 adds the Quit menu item to the File menu and sets the command to call the self.destroy() function. Finally, line 19 sets the menubar to be used as the main window's menu. The self.destroy() function will cause the application to quit.

If you run the application now, you will see that you can click on the Quit menu item on the File menu to quit the application. contain self as one of the parameters. When you are creating a GUI program that handles more than one process, it is useful to make use of tabbed windows. This allows you to put more of a user interface on each tab. For our application, we want to be able to handle two processes: 1) Set the database to query and 2) Run select queries on the chosen database. Therefore, we will make a tab for each process. Here is the modified code that starts to do that:

   def setDB(self):
      host = self.host_entry.get().strip()
      dbname = self.db_entry.get().strip()
      user = self.user_entry.get().strip()
      password = self.pass_entry.get().strip()
      self.conn_string = "host='" + host + "' dbname='"
      self.conn_string += dbname + "' user='" + user
      self.conn_string += "' password='" + password + "'"
      self.textarea.delete("1.0","end")
      try:
         self.conn = psycopg2.connect(self.conn_string)
         self.cursor = self.conn.cursor()
         sql = "select table_name,table_type from "
         sql += " information_schema.tables where " 
         sql += " table_schema='public'"
         self.cursor.execute(sql)
         results = self.cursor.fetchall()
         print(results)
         res_txt = "tables and views:\n"
         for result in results:
            res_txt += result[0] +", " 
            res_txt += result[1].replace('BASE ','') + "\n"
         self.textarea.insert(tk.END,res_txt)
      except psycopg2.DatabaseError as e:
         self.textarea.insert(tk.END,str(e))
         
   def runQuery(self):
      sql = self.query_area.get("1.0","end").strip()
      self.cursor.execute(sql)
      results = self.cursor.fetchall()
      headers = [desc[0] for desc in self.cursor.description]
      lines = []
      for result in results:
         line = []
         for item in result:
            line.append(item)
         lines.append(line)
      print(lines)
      self.updateTable(headers,lines)
      
   def updateTable(self,headers,lines):
      for child in self.tree.get_children():
         self.tree.delete(child)
      cols = []
      for header in headers:
         cols.append(header)
      print("cols:",cols)
      self.tree["columns"] = cols
      for i in range(0, len(cols)):
         self.tree.column(cols[i],width=700//len(cols))
      for header in headers:
         self.tree.heading(header,text=header)
      for line in lines:
         value_l = []
         for item in line:
            if (type(item) == str):
               item = item.strip()
            value_l.append(str(item))
         self.tree.insert("","end",text="",values=value_l)
      self.tree.grid(row=0, column=0, sticky='we')
      self.grid_columnconfigure(0, weight=1)
      self.tree.pack(fill=tk.BOTH, side=tk.TOP, expand=True)
