Web based text file processing using JavaScript

Processing a text file on a web page

Learning how to process a text file on a web page is a useful introduction to writing web applications using JavaScript. If you want to be able to write a web application you will need to be able to build the web front end, even if the back end is implemented using Node.js.

Save the following file, "readTextFile.html" in a directory that contains the file you want to process. 


<!DOCTYPE html>
<html>
   <head>
      <meta charset="utf-8">
      <script>
         document.addEventListener('DOMContentLoaded',init);
         function init() {
            console.log('init() called');
         }
      </script>
   </head>
   <body>
   </body>
</html>

Line 1 tells the browser to use HTML5. Line 2 and line 14 are the start and end tags for the <html> element. Lines 3-11 define the <head> element. This is where your JavaScript code should go. Line 4 makes it so that the UTF-8 character set is used to display all characters on the web page. That is a form of security as some malicious codes are prevented by specifying this character set. Lines 5-10 define a <script> element that holds JavaScript code.

Line 6 makes it so that when the body of the web page completes loading the init() function is called. You will probably see JavaScript code that is placed at the end of the <body> element to have this same effect. However, to put JavaScript code is considered by many (including your instructor) to be bad practice as it is better to keep your JavaScript code separate from the HTML markup in the <body> element.

Lines 7-9 define the init() function. At this point all this function does is print 'init() called' to the console on line 6.

If you open "readTextFile.html" in Chromium (or Chrome or Brave), hitting F12 will bring up the developer's console that allows you to see what is printed using console.log() . So, when you are doing JavaScript programming, you should have the developer's console open.

Firefox has changed its developer console so that it no longer displays console.log() output automatically. So, avoid using Firefox for JavaScript program development. Once your program works correctly, you can try it out on Firefox. The program should work, it is just that it will be easier to get the program to work if you use Chrome because using console.log() is an essential debugging tool.

Using the Node.js http-server package

When creating web applications in JavaScript, you will eventually need to have the web pages served out. Otherwise, web browsers like Chrome will encounter problems with accessing local files and also things like window.localStorage and IndexedDB. In addition, when we create a web application that has to connect to a database backend, the web front end must be served using a http server so that we can whitelist the request for the Node.js server. So, let's learn how to serve the pages using the Node.js package http-server . This package is installed on your virtual machine in lab, but if this is not on your Ubuntu machine, you can install it using the following command: 


npm install -g http-server

The -g switch will install this package globally so http-server can be run from any directory.

We need to start the http-server in the directory that contains all the files of interest. These are the files "readTextFile.html" and "courses.csv". Assume that these are in the directory ~/Desktop/text_processing . Then, perform the following steps: 


$ cd ~/Desktop/text_processing
$ http-server -a localhost -p 8000

The options that we will make use of are:

Option Description
-a Address to use (default: 0.0.0.0)
-p Port to use (default 8080)
--cors Enable CORS (common origin resource sharing)

Although we are not making use of the last option here, we may use this for a web application later. The settings used above will mean that the URL for the served pages will begin with: 


localhost:8000

So, this means we can see "readTextFile.html" by browsing to localhost:8000/readTextFile.html .

When you launch http-server, your terminal will look like this to start:

Starting up http-server, serving ./

Available on:

http://localhost:8000

Hit CTRL-C to stop the server

<!DOCTYPE html>
<html>
   <head>

The new lines are lines 7-9, lines 11-13 and lines 18-19. Let's talk about the changes to the HTML markup on lines 18-19 first. Line 18 has the <br> element removed so that the element defined on line 19 will be on the same line. Line 19 defines a <button> element that will be used to load the chosen file. Note that the <button> element has an id="loadFile" attribute that will be used to get a reference to this button.

Line 11 just comments out the printing of 'init() called" since we know that this is working. Line 12 gets a reference to the <button> element that we just defined on line 19. Line 13 sets things so that the handleLoadFile() function gets called when that button is clicked.

Lines 7-9 define the current version of the handleLoadFile() function. At this point, all this function does on line 8 is print 'handleLoadFile() called' to the console.

The following screen shot shows the new version of the application in the browser:

Note that the file "courses.csv" was browsed for and selected. Also, the Load File button was pressed so the console shows 'handleLoadFile() called'. Next, we want to put in code that will allow us to start processing the input file: 


<!DOCTYPE html>
<html>
   <head>
      <meta charset="utf-8">
      <script>
         document.addEventListener('DOMContentLoaded',init);
         function readUploadedFile(file) {
            const fileReader = new FileReader();
            let promise = new Promise((resolve, reject) => {
               fileReader.onerror = () => {
                  fileReader.abort();
                  reject(new DOMException("Problem reading"));
               };
               fileReader.onload = () => {
                  resolve(fileReader.result);
               };
               fileReader.readAsText(file);
            });
            return promise;
         }
        async function handleLoadFile() {
            //console.log('handleLoadFile() called');
            const chooseFile = document.getElementById('chooseFile');
            const file = chooseFile.files[0];
            let contents = await readUploadedFile(file);
            processFile(contents);
         }
         function processFile(contents) {
            console.log(contents);
         }
         function init() {
            //console.log('init() called');
            const loadFile = document.getElementById('loadFile');
            loadFile.addEventListener('click',handleLoadFile);
         }
      </script>
   </head>
   <body>
      <input id="chooseFile" type="file">
      <button id="loadFile">Load File</button><br>
      <textarea id="textarea" rows="5" cols="80"></textarea>
   </body>
</html>

The new lines are 7-20, lines 21-26 and lines 28-30. Lines 7-20 define the readUploadedFile() function. This file will take the file object from the chosen filename and return the contents of that file. The process of reading the file is an asynchronous process. This means, we need to make the program wait until the file has been read in. Otherwise, the contents of the read-in file will be undefined. Line 8 creates a FileReader object. Lines 9-18 define a Promise that will start the file reading process on line 17. If there is an error reading the file, lines 10-13 will handle that error and stop the file reading process. If there is no error and file is loaded, lines 14-16 will resolve the promise with the results of reading that file. Line 19 returns the promise so that the calling function can get the results.

Lines 21-26 show the changes to the handleLoadFile() function. Line 21 adds the await declaration to the function header. This allows the use of await inside the function. The purpose of await is to halt the JavaScript processing until the process that is being awaited completes. This allows an asynchronous process to be handled in a synchronous manner. With the addition of Promises, this has made asynchronous processing a lot more straightforward. Line 22 just comments out the printing to the console that is no longer needed. Line 23 gets a reference to the <input> element that is used to choose the file. Line 24 gets the file object from that <input> element. Line 25 uses await to call the readUploadedFile() function. When this process completes, the contents of the file is stored in the variable contents . Line 26 calls the processFile() function to actually process the contents.

Lines 28-30 define the processFile() function. This is the function that will do the actual processing of the input file. This function needs to be modified if you are reading in a different kind of input file.

Let's modify processFile() to do some very simple processing: 


<!DOCTYPE html>
<html>
   <head>
      <meta charset="utf-8">
      <script>
         document.addEventListener('DOMContentLoaded',init);
         function readUploadedFile(file) {
            const fileReader = new FileReader();
            let promise = new Promise((resolve, reject) => {
               fileReader.onerror = () => {
                  fileReader.abort();
                  reject(new DOMException("Problem reading"));
               };
               fileReader.onload = () => {
                  resolve(fileReader.result);
               };
               fileReader.readAsText(file);
            });
            return promise;
         }
         async function handleLoadFile() {
            //console.log('handleLoadFile() called');
            const chooseFile = document.getElementById('chooseFile');
            const file = chooseFile.files[0];
            let contents = await readUploadedFile(file);
            processFile(contents);
         }
         function processFile(contents) {
            // This is the function that you modify
            let data = contents.split("\n");
            let out = "";
            for (let i = 0; i < data.length; i++) {
               out += data[i] + "\n";
            }
            const textarea = document.getElementById('textarea');
            textarea.value = out;
         }
         function init() {
            //console.log('init() called');
            const loadFile = document.getElementById('loadFile');
            loadFile.addEventListener('click',handleLoadFile);
         }
      </script>
   </head>
   <body>
      <input id="chooseFile" type="file">
      <button id="loadFile">Load File</button><br>
      <textarea id="textarea" rows="5" cols="80"></textarea>
   </body>
</html>

The new lines are lines 29-36. Line 29 just comments out a print statement that is no longer needed. Line 30 splits the contents into lines from the file and stores this in an array called data . Lines 31-34 create a string called out that has each line from data separated by a newline character. Line 35 gets a reference to the <textarea> element that we will use for output. Line 36 sets the value of the <textarea> element to be out .

Note that the textarea was set up with only 5 rows, so you see the first 5 lines from the input file. You could just select everything in the textarea and paste that into an output file. But, let's make things a little fancier by creating a link to a file that can be downloaded: 


<!DOCTYPE html>
<html>
   <head>
      <meta charset="utf-8">
      <script>
         document.addEventListener('DOMContentLoaded',init);
         function readUploadedFile(file) {
            const fileReader = new FileReader();
            let promise = new Promise((resolve, reject) => {
               fileReader.onerror = () => {
                  fileReader.abort();
                  reject(new DOMException("Problem reading"));
               };
               fileReader.onload = () => {
                  resolve(fileReader.result);
               };
               fileReader.readAsText(file);
            });
            return promise;
         }
         async function handleLoadFile() {
            //console.log('handleLoadFile() called');
            const chooseFile = document.getElementById('chooseFile');
            const file = chooseFile.files[0];
            let contents = await readUploadedFile(file);
            processFile(contents);
         }
         function processFile(contents) {
            // This is the function that you modify
            let data = contents.split("\n");
            let out = "";
            for (let i = 0; i < data.length; i++) {
               out += data[i] + "\n";
            }
            const textarea = document.getElementById('textarea');
            textarea.value = out;
            let out_data = new Blob([out], {
               type: 'text/plain'
            });
            let url = window.URL.createObjectURL(out_data);
            const download = document.getElementById('download');
            download.href = url;
         }
         function init() {
            //console.log('init() called');
            const loadFile = document.getElementById('loadFile');
            loadFile.addEventListener('click',handleLoadFile);
         }
      </script>
   </head>
   <body>
      <input id="chooseFile" type="file">
      <button id="loadFile">Load File</button><br>
      <a id="download" href=""
         download="out.txt">Download output</a><br>
      <textarea id="textarea" rows="5" cols="80"></textarea>
   </body>
</html>

The new lines are lines 37-42 and lines 54-55. Let's look at the changes to the HTML markup first on lines 54-55. Lines 54-55 define an <a> (anchor) element that will be used to create a link to a downloadable output file. Note that the href attribute is currently blank but must be set before the user clicks on this link. That will be taken care of when we process the input file. The download attribute is set to "out.txt". This will be the default name for the output file, but the user can override this when saving the output file. The id="download" attribute will be used to get a reference to this <a> so that we can set the value for the href attribute.

Lines 37-42 are the additional lines inside the processFile() function. Lines 37-39 create a Blob object that is a plain text file. Line 40 uses the window.URL.createObjectURL() function to create a URL for this new data object. Line 41 gets a reference to the <a> element that is a link to the file to be downloaded. Line 42 sets the href attribute to the newly created URL (that is constructed for the Blob object).

When you reload the application in the browser, you select the input file. After you load that file, you can click on the Download output link to save the output.

Steps to make use of this application

  1. Copy the last version of the code above to a suitable directory for the file you are processing.
  2. You may want to modify the number of rows for the <textarea> element so that you can see more lines of output without scrolling.
  3. Modify the processFile() function so that it processes the input file contents for the given situation.
  4. When the output in the <textarea> is correct, download the output and save the file.
