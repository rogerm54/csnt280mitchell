Using Docker

A separate Docker container for PostgreSQL

Although we have set up a way to connect a PostgreSQL docker container and a PHP-Apache docker container using docker-compose, let's create another PostgreSQL docker container for projects that don't require PHP to work with PostgreSQL.

Start by going to your Desktop and creating a directory called docker_postgres . Inside that directory create a directory called docs . Here are the commands to do this: 


$ cd ~/Desktop
$ mkdir -p docker_postgres/docs

Next, we want to create a file called Dockerfile inside the docker_postgres directory. You can use the geany editor to do this. Here are the commands: 


$ cd ~/Desktop/docker_postgres
$ geany Dockerfile &

Within the geany editor, type the following commands: 


FROM postgres
RUN apt-get update -y && apt-get install nano -y
WORKDIR /docs

Line 1 will pull the latest postgres package from Docker Hub. Line 2 will run update the repositories and install nano (a simple text editor). Line 3 will set the working directory to be /docs . We will use /docs as a persistent volume for our Docker container.

Building the Docker image

Now that we have Dockerfile set up, let's build the Docker image. We will call this image "postgres_img". Here are the commands: 


$ cd ~/Desktop/docker_postgres
$ docker build -t postgres_img .

Make sure you type the dot (.) at the end of the build command. This will cause Docker to look inside the current directory for the file "Dockerfile" and build the image based on the contents of "Dockerfile". The name of the image will be "postgres_img".

This will take some time to run as it involves downloading the latest postgres package, and then updating the repositories to add nano . Once it completes, you should be able to see the image by running: 


$ docker image ls

This will display the following: 


REPOSITORY                        TAG                 IMAGE ID            CREATED             SIZE
postgres_img                      latest              ef94d2d51604        49 minutes ago      333MB

Other Docker images may exist because of the containers built using docker-compose, but they are not shown here for brevity. Also, note that I built my image 49 minutes before making this page.

Creating the Docker container

Next we want to create the Docker container from the Docker image we just built. You can create more than one Docker container from a Docker image. If you think of the Docker image like a class definition, a Docker container is like a running instance of that class. We will give our container a name and reuse that container by stopping and starting the container. This will allow us to have changes that we make inside that container persist.

When we create the container, we are going to use a named persistent volume. A named persistent volume is a way of connecting a shared directory between the host operating system and the Docker container. This will allow us to create and edit files in the host operating system and have those files get mapped inside the Docker container. With this in mind, here are the commands to build and initially run the Docker container: 


$ cd ~/Desktop/docker_postgres
$ docker container run -d --name=postgres_container -v $(pwd)/docs:/docs postgres_img

The command starts off with: 


docker container run

which is used to run a Docker container. The -d switch means to run the container as a daemon in the background. This part of the command: 


--name=postgres_container

names the Docker container. This part of the command: 


-v $(pwd)/docs:/docs

will create a named persistent volume that is mapped to the Docker container. The $(pwd) will call the print working directory command. If you don't want to use that command, you could change that part of the command to: 


-v ~/Desktop/docker_postgres/docs:/docs

Finally, the last part of the command: 


postgres_img

is the name of the image we want to use to create the container. Here is what the command looks like when it succeeds in creating and running the container: 


$ docker container run -d --name=postgres_container -v $(pwd)/docs:/docs postgres_img
248048aee1e8100cb0b7d126425306ea29b51216be9f7a8cc584e2385a878c40

That is, you will see a long hexstring that is the id string for the running Docker container. To verify that this container is running, use the following: 


$ docker container ls
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
248048aee1e8        postgres_img        "docker-entrypoint.s…"   2 minutes ago       Up 2 minutes        5432/tcp            postgres_container

Stopping and restarting the container

To stop the Docker container, run the following command: 


$ docker stop postgres_container
postgres_container

As is shown, the name of the container that has been stopped is shown on the next line. To restart the container, run the following: 


$ docker start postgres_container
postgres_container

Once again, you can verify that the container is running by doing this: 


$ docker container ls
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
248048aee1e8        postgres_img        "docker-entrypoint.s…"   5 minutes ago       Up 19 seconds       5432/tcp            postgres_container

Entering the container

To enter the docker container just run the following command: 


$ docker exec -it postgres_container bash
root@32bd083594ed:/docs#

Now you can run whatever you need to from this root prompt inside the container. For a first time setup of the PostgreSQL server, complete the following section:

First time setup of a PostgreSQL Docker container

Get a root prompt inside the PostgreSQL docker container and use the postgres account to create a superuser called csnt280man . Here are the steps to follow: 


$ cd ~/Desktop/docker_postgres
$ docker start postgres_container
$ docker exec -it postgres_container bash
root@32bd083594ed:/docs# su postgres
postgres@32bd083594ed:/docs$ psql postgres
psql (10.5 (Debian 10.5-1.pgdg90+1))
Type "help" for help.

postgres=# create role csnt280man login createdb createrole;
CREATE ROLE
postgres=# alter role csnt280man encrypted password 'pgsql_man';
ALTER ROLE
postgres=# create database csnt280db owner csnt280man;
CREATE DATABASE
postgres=# \q
postgres@32bd083594ed:/docs$ exit
exit
root@32bd083594ed:/docs#

Line 1 just changes into the correct directory. Line 2 should be run if the Docker container is not already started. Line 3 gets the root prompt inside the Docker container. Line 4 switches the user to the postgres user. Line 5 runs the psql command with the user set to postgres. Since the administrator database is also named postgres and PostgreSQL is configured to use trust authentication, line 5 will get you into the "postgres" database as can be seen on line 9.

On line 9, we create the superuser csnt280man . On line 11, we set the password for the csnt280man user. On line 13, we create a database called csnt280man for the user with the same name. Line 15 exits psql and line 16 exits the switch to the postgres user, so that we go back to the root prompt.

Removing a security weakness

With the way the PostgreSQL is configured at this point, a user can log into any database without supplying a password. To prevent this, we need to change the configuration inside of /var/lib/postgresql/data/pg_hba.conf. Here are the steps to do this:

At the root prompt run the following command: 


root@32bd083594ed:/docs# nano /var/lib/postgresql/data/pg_hba.conf

When nano opens this file type ALT-G, then enter 84. This will take you to line 84 in the file. You should comment that line using a # symbol and add the following line: 


local      add      add      md5

The resulting file should look like this: 


# TYPE DATABASE        USER            ADDRESS                 METHOD

# "local" is for Unix domain socket connections only
#local   all             all                                     trust
local   all     all     md5
# IPv4 local connections:

The line numbers shown here are not the line numbers from the file. What is line 4 here is actually line 84 in the file. That is the line that should be commented out. What is shown as line 5 here is the line that replaces the commented out line.

Save the file by typing CTRL-O and hitting <enter>. Then, type CTRL-X to exit nano. Now, we need to exit the Docker container, stop the Docker container and start the Docker container again. Then, we can use the csnt280man user to administer the database: 


root@32bd083594ed:/docs# nano /var/lib/postgresql/data/pg_hba.conf 
root@32bd083594ed:/docs# exit
exit
$ docker stop postgres_container
postgres_container
$ docker start postgres_container
postgres_container
$ docker exec -it postgres_container bash
root@32bd083594ed:/docs#

Line 1 shows that we had used nano to edit the configuration file. Line 2 shows us exiting the root prompt (leaving the Docker container). Line 4 stops the Docker container. Line 6 starts up the Docker container again. Stopping and then starting the Docker container allows the configuration change (made in pg_hba.conf) to take place. Line 8 gets us back inside the Docker container.

From that root prompt, we will use the csnt280man account to create a user named bob and create a database called bookstore_db that is owned by bob. Here are the steps: 


root@32bd083594ed:/docs# psql -U csnt280man
Password for user csnt280man: 
psql (10.5 (Debian 10.5-1.pgdg90+1))
Type "help" for help.

csnt280man=> create role bob login;
CREATE ROLE
csnt280man=> alter role bob encrypted password 'somepass';
ALTER ROLE
csnt280man=> grant bob to csnt280man;
GRANT ROLE
csnt280man=> create database bookstore_db owner bob;
CREATE DATABASE
csnt280man=> \q
root@32bd083594ed:/docs#

On line 1 we start psql. Since we don't supply the database name, psql will assume the database has the same name as the user. So, on line 6, you can see that the database being used is csnt280man . You can see this because of this prompt: 


csnt280man=>

Line 6 is used to create a regular user named bob . Line 8 sets the password for bob . Line 10 makes it so that csnt280man can grant ownership of a database to bob. On line 12, we create the database bookstore_db and set the owner of that database to be bob . Line 14 cause the csnt280man user to quit using psql.

We will make use of this database later in this lesson .

Rebuilding the docker-compose containers

On the images, there are two Docker containers that were set up using the docker-compose command and the configuration files stored in the folder ~/Desktop/docker_cent280. If you want to rebuild those containers (and wipe out all the Docker images), you can run the following command: 


$ docker system prune -a

You should only do this if you want to remove every Docker image, every Docker container and every Docker persistent volume. I will do this if I want to start with a clean set of images and containers.

If you clean out everything, you can follow these steps to rebuild the containers that you already had on the images for the course: 


$ cd ~/Desktop/docker_cent280
$ docker-compose up -d

From that point forward, you can stop those Docker containers by doing this: 


$ cd ~/Desktop/docker_cent280
$ docker-compose stop

To start those Docker containers you do this: 


$ cd ~/Desktop/docker_cent280
$ docker-compose start

To enter into the postgresql container, you would do this: 


$ cd ~/Desktop/docker_cent280
$ docker-compose exec postgresql bash

To enter into the php-apache container, you would do this: 


$ cd ~/Desktop/docker_cent280
$ docker-compose exec php-apache bash
