Objective 1 - Inserting data into a relational database

To gain more experience with inserting data into a relational database, let's use another example. For this example, we are creating a simple database that keeps track of the customers for a bookstore. The database will be called "bookstore_db". The steps to setup a Docker container and create this database are shown here .

Change into the ~/Desktop/docker_postgres/docs directory and create the file "bookstore_init.sql" with the contents shown below. 


-- bookstore_init.sql
drop table if exists books cascade;
drop table if exists customers cascade;
drop table if exists sales cascade;

create table books (
   id serial,
   title varchar(100),
   author varchar(100),
   price decimal(6,2),
   primary key (id)
);

create table customers (
   id serial,
   first_name varchar(30),
   last_name varchar(40),
   primary key (id)
);

create table sales (
   id serial,
   customer_id integer references customers(id),
   book_id integer references books(id),
   primary key (id)
);

create or replace view sales_view as
   select customers.id as cid, customers.first_name,
   customers.last_name, books.id as bk_id,
   books.title, books.author, books.price from
   customers join sales on customers.id=sales.customer_id 
   join books on sales.book_id=books.id;

Check to see if the postgres_container is running by typing: 


docker container ls

If that container is not running, start it up using: 


docker start postgres_container

To run the "bookstore_db.sql" script, start by entering into the postgres_container container: 


$ cd ~/Desktop/docker_postgres
$ docker exec -it postgres_container bash

This will open up a root prompt inside the /docs directory. Check to see that "bookstore_init.sql" is in that directory:

Once you have verified this, use psql to connect to the database as the 'bob' user:

Inserting data into the bookstore_db database

Let's look at using the SQL INSERT statement to insert a customer into the customers table. You could do this: 


insert into customers (first_name, last_name) values 
   ('Jane','Doe');

The first set of parentheses specifies the list of field names you want to insert values into, and the second set of parentheses specifies the list of values you want to insert. You need to make sure that there is one value for each field and that the values are in the same order as the order for the fields.

Note that the id field is not specified. This is because the id field has a data type of serial . That means that if the value for the id is not supplied, the next value from the the "customers_id_seq" sequence is inserted by default. This is a good way to insert a record when you don't need to specify the id value. An alternative method for inserting this same record would be: 


insert into customers (id, first_name, last_name) values 
   (1, 'Jane', 'Doe');

This sets the id value explicity to 1. Specifying the id this way can be useful when setting up test data. Keep in mind, that if the id already exists, the insert will fail.

Also, note that the value of a text or any other string variable in PostgreSQL, must be enclosed in single quotes. You cannot use double quotes for a string value in PostgreSQL.

Let's create the file "add_initial_data.sql" inside the ~/Desktop/docker_postgres_docs directory: 


$ cd ~/Desktop/docker_postgres/docs
$ geany add_initial_data.sql &

Use the following contents: 


insert into customers (id, first_name,last_name) values
   (1, 'Jane','Doe');
insert into customers (id, first_name,last_name) values
   (2, 'John','Doe');
insert into customers (id, first_name,last_name) values
   (3, 'Bill','Gates');
   
insert into books (id, title,author,price) values
   (1, 'Go Dog Go','P.D. Eastman',7.99);
insert into books (id, title,author,price) values
   (2, 'The Cat in the Hat','Dr. Seuss',8.99);
insert into books (id, title,author,price) values
   (3, 'Ender''s Game','Orson Scott Card',9.99);

insert into sales (customer_id,book_id) values (1,1);
insert into sales (customer_id,book_id) values (1,2);
insert into sales (customer_id,book_id) values (2,1);
insert into sales (customer_id,book_id) values (2,3);
insert into sales (customer_id,book_id) values (3,1);
insert into sales (customer_id,book_id) values (3,2);
insert into sales (customer_id,book_id) values (3,3);

Lines 1-6 insert records into the customers table. Lines 8-13 insert records into the books table. Lines 15-21 insert records into the sales table. By specifying the id values for the customers and books records, it is easy to see what the insert statements for the sales table are doing. For example, on line 15, the customer_id of 1 and book_id of 1 means that Jane Doe bought Go Dog Bo.
